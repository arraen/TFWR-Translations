# Simulation (Симуляція)

Симуляції дозволяють швидко тестувати код без зміни стану реальної ферми.
Початковий стан симуляції можна вибрати вільно, і коли симуляція закінчується, реальна ферма буде в тому ж стані, в якому вона була до початку симуляції.

Функція `simulate()` використовується для запуску симуляції.

файл, в якому має початися виконання
`filename = "f1"`

почати з усім розблокованим і повністю покращеним
`sim_unlocks = Unlocks`

почати з 10000 моркви та 50 сіна
`sim_items = {Items.Carrot : 10000, Items.Hay : 50}`

почати з глобальною змінною "a" зі значенням 13
`sim_globals = {"a" : 13}`

використовувати фіксований випадковий seed
`seed = 0`

прискорити симуляцію на коефіцієнт 64
`speedup = 64`

запустити симуляцію
`run_time = simulate(filename, sim_unlocks, sim_items, sim_globals, seed, speedup)`

Функція `simulate()` повертає час у секундах, який знадобився для симуляції заданого стартового файлу.

### File Name (Ім'я файлу)
Перший аргумент функції simulate — це ім'я файлу. Це ім'я, яке відображається у верхній частині вікна коду. Симуляція запустить вказаний файл так, ніби ви натиснули кнопку Execute на ньому.

### Starting Unlocks (Початкові розблокування)
Усі функції програмування, такі як цикли, if statements, списки, словники,... завжди залишатимуться розблокованими.

Другий аргумент дозволяє вказати, які unlocks/upgrades симуляція повинна мати на старті на додаток до функцій програмування. Це має бути послідовність (sequence) unlocks. Симуляція почнеться з усіма unlocks у послідовності, покращеними до їх максимального рівня.

Якщо ви хочете вказати рівень покращення, відмінний від максимального, ви можете передати словник, який зіставляє unlocks з рівнями розблокування. У цьому випадку від'ємні значення відповідають максимальному рівню розблокування.

### Starting Items (Початкові предмети)
Третій аргумент дозволяє передати словник, який зіставляє items з числами. Він визначає предмети, з якими почнеться симуляція.

### Starting Globals (Початкові глобальні змінні)
Оскільки симуляція запускає виконання абсолютно нової програми, ви не можете отримати доступ до змінних з програми, яка запускає симуляцію.
Однак можна передати значення в симуляцію, використовуючи четвертий аргумент. Це dict, який зіставляє імена змінних у вигляді рядків зі значеннями. Ці змінні потім додаються до глобальної області видимості (global scope) виконання всередині симуляції.

Зауважте, що це копіює всі значення, тому їх зміна (mutating) всередині симуляції не вплине на оригінальні значення поза симуляцією. Неможливо повернути значення з симуляції, крім часу, який вона зайняла.

### Random Seed (Випадкове зерно)
П'ятий аргумент дозволяє вказати random seed, що використовується в симуляції. Це має бути позитивне ціле число. Від'ємні значення призведуть до використання випадкового seed.

Random seed впливає на все: від часу росту рослин до макетів лабіринтів і часу розпаду води. Якщо ви запускаєте одну й ту саму симуляцію кілька разів з тим самим random seed і тими ж початковими умовами, результат завжди має бути однаковим.

### Speedup (Прискорення)
Шостий аргумент — це початкове прискорення симуляції. Це дозволяє швидко тестувати речі. Якщо гра не зможе підтримувати встановлену швидкість, вона сповільниться автоматично.

Speedup жодним чином не впливає на результат симуляції. Він існує лише для того, щоб зменшити час очікування.
